Issue tracking-significa rastreamento de problemas, monitoramento de bugs, ou acompanhamento de pendências. É o processo de registrar, priorizar e resolver erros ou tarefas (tickets) em desenvolvimento de software e atendimento ao cliente, garantindo que nenhum item se perca. 




"Pull request"-(frequentemente abreviado como PR) pode ser traduzido no contexto de desenvolvimento de software como solicitação de envio, solicitação de extração ou pedido de mesclagem. É o mecanismo utilizado no Git/GitHub para notificar membros da equipe sobre alterações concluídas em uma branch, solicitando revisão e integração ao código principal.
